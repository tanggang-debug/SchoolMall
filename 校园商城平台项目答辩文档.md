## 一、项目介绍

### 1.1 项目背景与目标

#### 项目背景

随着校园电商需求的日益增长，传统的线下交易方式存在信息不透明、交易效率低、管理困难等问题。本项目旨在构建一个面向高校师生的小型电商平台——**校园商城平台**，为学生提供便捷的购物体验，为商户提供高效的销售渠道，为管理员提供完善的监管工具。

#### 项目目标

**核心目标**：实现一个完整的电商业务闭环

```
注册登录 → 浏览商品 → 加入购物车 → 创建订单 → 模拟支付 → 商户发货 → 确认收货 → 评价
```

**具体目标**：
1. **多角色支持**：学生、商户、管理员三种角色，各有独立权限和功能
2. **完整业务流程**：覆盖商品发布、审核、上架、下单、支付、发货、收货、评价全过程
3. **技术实践**：应用微服务架构、前后端分离、JWT认证、Redis缓存、状态机等现代技术
4. **可演示性**：提供完整的前端界面和后端接口，可直接运行演示

| 模块 | 学生 | 商户 | 管理员 |

**1. 学生购物流程**：

```
登录 → 浏览商品 → 查看详情 → 加入购物车 → 确认订单 → 创建支付 →

```


### 2.1 前端架构展示

#### Vue 3 + Composition API

前端使用 Vue 3 的 Composition API 组织代码，代码逻辑更清晰、复用性更好。

**示例：用户状态管理（stores/user.js）**

```javascript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useUserStore = defineStore('user', () => {
  // 状态
  const token = ref(localStorage.getItem('token') || '')
  const userId = ref(localStorage.getItem('userId') || '')
  const username = ref(localStorage.getItem('username') || '')
  const role = ref(Number(localStorage.getItem('role') || 0))

  // 计算属性
  const isLogin = computed(() => !!token.value)
  const isAdmin = computed(() => role.value === 3)
  const isMerchant = computed(() => role.value === 2)

  // 方法
  function setUser(data) {
    token.value = data.token
    userId.value = data.userId
    username.value = data.username
    role.value = data.role
    // 持久化到 localStorage
    localStorage.setItem('token', data.token)
    localStorage.setItem('userId', data.userId)
    localStorage.setItem('username', data.username)
    localStorage.setItem('role', data.role)
  }

  function logout() {
    token.value = ''
    userId.value = ''
    username.value = ''
    role.value = 0
    localStorage.clear()
  }

  return { token, userId, username, role, isLogin, isAdmin, isMerchant, setUser, logout }
})
```

**技术亮点**：
1. 使用 Pinia 替代 Vuex，API更简洁
2. 使用 Composition API 的 ref 和 computed，代码组织灵活
3. 自动持久化到 localStorage，刷新页面不丢失登录状态

#### Axios 请求封装与拦截器

前端统一封装 Axios，添加请求拦截器（自动携带Token）和响应拦截器（统一错误处理）。

**示例：request.js**

```javascript
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '../stores/user'

const request = axios.create({ baseURL: '/api', timeout: 15000 })

// 请求拦截器：自动携带 JWT Token
request.interceptors.request.use(config => {
  const user = useUserStore()
  if (user.token) {
    config.headers.Authorization = `Bearer ${user.token}`
  }
  return config
})

// 响应拦截器：统一处理错误
request.interceptors.response.use(
  res => {
    const data = res.data
    if (data.code !== 200) {
      ElMessage.error(data.message || '请求失败')
      return Promise.reject(data)
    }
    return data
  },
  err => {
    ElMessage.error(err.response?.data?.message || '网络错误')
    return Promise.reject(err)
  }
)

export default request
```

**技术亮点**：
1. **请求拦截器**：自动在所有请求头中添加 `Authorization: Bearer ${token}`，无需手动处理
2. **响应拦截器**：统一判断业务码（code=200为成功），统一错误提示
3. **BaseURL配置**：所有请求统一走 `/api` 前缀，通过网关转发

#### 路由权限控制

前端使用 Vue Router 的路由守卫（beforeEach）实现权限控制。

**示例：router/index.js**

```javascript
const routes = [
  { path: '/login', component: () => import('../views/Login.vue') },
  {
    path: '/',
    component: () => import('../layouts/MainLayout.vue'),
    children: [
      { path: '', name: 'home', component: () => import('../views/Home.vue') },
      { path: 'product/:id', component: () => import('../views/ProductDetail.vue') },
      // 需要登录的路由：meta.auth = true
      { path: 'cart', component: () => import('../views/Cart.vue'), meta: { auth: true } },
      { path: 'orders', component: () => import('../views/OrderList.vue'), meta: { auth: true } },
      // 需要特定角色的路由：meta.role = 角色
      { path: 'merchant/products', component: () => import('../views/merchant/ProductManage.vue'), 
        meta: { auth: true, role: 2 } },
      { path: 'admin/users', component: () => import('../views/admin/UserManage.vue'), 
        meta: { auth: true, role: 3 } }
    ]
  }
]

// 全局路由守卫：权限控制
router.beforeEach((to, from, next) => {
  const user = useUserStore()
  
  // 需要登录但未登录：跳转到登录页
  if (to.meta.auth && !user.isLogin) {
    return next('/login')
  }
  
  // 需要特定角色但角色不匹配：跳转到首页
  if (to.meta.role !== undefined && user.role !== to.meta.role && user.role !== 3) {
    return next('/')
  }
  
  next()
})
```

**技术亮点**：
1. **meta元信息**：在路由配置中声明需要登录（auth: true）和需要角色（role: 2/3）
2. **全局守卫**：每次路由跳转前自动检查权限，未登录跳转登录页，角色不匹配跳转首页
3. **管理员特权**：管理员（role=3）可以访问所有页面

### 2.2 后端架构展示

#### Spring Boot RESTful API设计

后端采用标准的 RESTful API 设计风格，每个模块独立Controller。

**示例：商品Controller（ProductController.java）**

```java
@RestController
@RequestMapping("/api/products")
@RequiredArgsConstructor
public class ProductController {

    private final ProductService productService;

    // GET /api/products - 商品列表（分页、搜索）
    @GetMapping
    public Result<PageResult<ProductResponse>> list(
            @RequestParam(defaultValue = "1") int page,
            @RequestParam(defaultValue = "10") int size,
            @RequestParam(required = false) String keyword,
            @RequestParam(required = false) Long categoryId,
            @RequestParam(required = false) Integer status) {
        return Result.ok(productService.list(page, size, keyword, categoryId, status));
    }

    // GET /api/products/{id} - 商品详情
    @GetMapping("/{id}")
    public Result<ProductResponse> getById(@PathVariable Long id) {
        return Result.ok(productService.getById(id));
    }

    // POST /api/products - 创建商品（需登录）
    @PostMapping
    @RequireLogin
    public Result<ProductResponse> create(@Valid @RequestBody ProductRequest request) {
        return Result.ok(productService.create(request));
    }

    // PUT /api/products/{id} - 更新商品（需登录）
    @PutMapping("/{id}")
    @RequireLogin
    public Result<ProductResponse> update(@PathVariable Long id, 
                                          @Valid @RequestBody ProductRequest request) {
        return Result.ok(productService.update(id, request));
    }

    // PUT /api/products/{id}/audit - 审核商品（需管理员权限）
    @PutMapping("/{id}/audit")
    @RequireRole(3)
    public Result<ProductResponse> audit(@PathVariable Long id, 
                                         @Valid @RequestBody ProductAuditRequest request) {
        return Result.ok(productService.audit(id, request));
    }
}
```

**技术亮点**：
1. **RESTful风格**：GET查询、POST创建、PUT更新、DELETE删除
2. **统一响应格式**：所有接口返回 `Result<T>`，包含 code、message、data
3. **权限注解**：`@RequireLogin` 需要登录，`@RequireRole(3)` 需要管理员角色
4. **参数校验**：`@Valid` + `@NotBlank`、`@NotNull` 等注解自动校验参数

#### JWT认证实现

后端使用 JWT（JSON Web Token）实现无状态认证，Token包含用户ID、用户名、角色、过期时间。

**示例：认证服务（AuthService.java）**

```java
@Service
@RequiredArgsConstructor
public class AuthService {

    private final UserMapper userMapper;
    private final JwtTokenProvider jwtTokenProvider;
    private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();

    @Transactional
    public AuthResponse login(LoginRequest request, HttpServletRequest httpRequest) {
        // 1. 查询用户
        User user = userMapper.selectOne(new LambdaQueryWrapper<User>()
                .eq(User::getUsername, request.getUsername()));
        
        if (user == null) {
            throw new BusinessException(ErrorCode.USER_NOT_FOUND, "用户不存在");
        }
        
        // 2. 校验密码（BCrypt加密）
        if (!passwordEncoder.matches(request.getPassword(), user.getPassword())) {
            throw new BusinessException(ErrorCode.PASSWORD_ERROR, "密码错误");
        }
        
        // 3. 生成 JWT Token
        String token = jwtTokenProvider.createToken(user.getId(), user.getUsername(), user.getRole());
        
        // 4. 返回响应
        AuthResponse resp = new AuthResponse();
        resp.setToken(token);
        resp.setUserId(user.getId());
        resp.setRole(user.getRole());
        resp.setUsername(user.getUsername());
        return resp;
    }

    @Transactional
    public AuthResponse register(RegisterRequest request) {
        // 1. 检查用户名是否已存在
        Long count = userMapper.selectCount(new LambdaQueryWrapper<User>()
                .eq(User::getUsername, request.getUsername()));
        if (count > 0) {
            throw new BusinessException(ErrorCode.FORBIDDEN, "用户名已存在");
        }
        
        // 2. 创建用户（BCrypt加密密码）
        User user = new User();
        user.setUsername(request.getUsername());
        user.setPassword(passwordEncoder.encode(request.getPassword()));
        user.setPhone(request.getPhone());
        user.setRole(request.getRole() != null ? request.getRole() : UserRole.STUDENT.getCode());
        userMapper.insert(user);
        
        // 3. 生成 JWT Token（注册成功自动登录）
        String token = jwtTokenProvider.createToken(user.getId(), user.getUsername(), user.getRole());
        
        AuthResponse resp = new AuthResponse();
        resp.setToken(token);
        resp.setUserId(user.getId());
        resp.setRole(user.getRole());
        resp.setUsername(user.getUsername());
        return resp;
    }
}
```

**技术亮点**：
1. **BCrypt密码加密**：使用 BCryptPasswordEncoder 加密密码，安全性高（加盐、不可逆）
2. **JWT Token生成**：包含用户ID、用户名、角色，过期时间2小时
3. **无状态认证**：Token自包含信息，无需服务端存储session，适合分布式系统

```java
@Service
@RequiredArgsConstructor
public class CartService {

    private static final String CART_KEY_PREFIX = "cart:";
    private final StringRedisTemplate redisTemplate;
    private final ObjectMapper objectMapper;
    private final ProductService productService;

    // 添加商品到购物车（Redis缓存）
    public void addItem(CartItemRequest request) {
        Long userId = requireUserId();
        
        // 1. 校验商品是否在售
        productService.requireOnSaleProduct(request.getProductId());
        
        // 2. 从 Redis 获取购物车数据
        String key = cartKey(userId);  // "cart:123"
        String field = String.valueOf(request.getProductId());
        String existing = (String) redisTemplate.opsForHash().get(key, field);
        
        // 3. 更新购物车数据
        CartEntry entry;
        if (existing != null) {
            // 商品已存在：累加数量
            entry = parseEntry(existing);
            entry.setQuantity(entry.getQuantity() + request.getQuantity());
        } else {
            // 商品不存在：新建条目
            entry = new CartEntry();
            entry.setQuantity(request.getQuantity());
            entry.setSelected(true);
        }
        
        // 4. 存入 Redis（Hash结构）
        redisTemplate.opsForHash().put(key, field, toJson(entry));
    }
    
    private String cartKey(Long userId) {
        return CART_KEY_PREFIX + userId;
    }
}
```

```java
@Service
@RequiredArgsConstructor
public class OrderService {

    private final OrderMapper orderMapper;
    private final OrderItemMapper orderItemMapper;
    private final ProductService productService;
    private final CartService cartService;

    @Transactional
    public List<OrderResponse> create(CreateOrderRequest request) {
        Long userId = UserContext.getUserId();
        
        // 1. 获取购物车已选商品
        Map<Long, CartEntry> selectedItems = cartService.getSelectedItems();
        if (selectedItems.isEmpty()) {
            throw new BusinessException(ErrorCode.CART_EMPTY, "购物车为空");
        }
        
        // 2. 校验商品库存
        for (Long productId : selectedItems.keySet()) {
            Product product = productService.requireProduct(productId);
            CartEntry entry = selectedItems.get(productId);
            if (product.getStock() < entry.getQuantity()) {
                throw new BusinessException(ErrorCode.STOCK_NOT_ENOUGH, 
                    "商品[" + product.getTitle() + "]库存不足");
            }
        }
        
        // 3. 创建订单
        Order order = new Order();
        order.setOrderNo(BizNoUtil.orderNo());  // "ORD202607010001"
        order.setUserId(userId);
        order.setAddressId(request.getAddressId());
        order.setStatus(OrderStatus.UNPAID.getCode());
        order.setRemark(request.getRemark());
        orderMapper.insert(order);
        
        // 4. 创建订单项、扣减库存
        BigDecimal totalAmount = BigDecimal.ZERO;
        for (Long productId : selectedItems.keySet()) {
            Product product = productService.requireProduct(productId);
            CartEntry entry = selectedItems.get(productId);
            
            // 创建订单项
            OrderItem item = new OrderItem();
            item.setOrderId(order.getId());
            item.setProductId(productId);
            item.setQuantity(entry.getQuantity());
            item.setPrice(product.getPrice());
            item.setTitle(product.getTitle());
            item.setImage(product.getImages());
            orderItemMapper.insert(item);
            
            // 扣减库存
            productService.deductStock(productId, entry.getQuantity());
            
            totalAmount = totalAmount.add(product.getPrice().multiply(new BigDecimal(entry.getQuantity())));
        }
        
        // 5. 更新订单总金额
        order.setTotalAmount(totalAmount);
        order.setPayAmount(totalAmount);
        orderMapper.updateById(order);
        
        // 6. 清空购物车已选商品
        cartService.removeItems(new ArrayList<>(selectedItems.keySet()));
        
        // 7. 返回订单信息
        return Collections.singletonList(toResponse(order));
    }
}
```

**技术亮点**：
1. **事务控制**：`@Transactional` 保证订单创建、库存扣减、购物车清空一致性
2. **库存校验**：创建订单前校验库存，避免超卖
3. **订单号生成**：使用时间戳生成唯一订单号（ORD + yyyyMMdd + 序号）
4. **状态初始化**：订单初始状态为 `UNPAID`（未支付）
5. **购物车同步**：下单成功后自动清空购物车已选商品

```java
@Service
@RequiredArgsConstructor
public class PaymentService {

    private final PaymentRecordMapper paymentRecordMapper;
    private final OrderService orderService;

    @Transactional
    public PaymentResponse create(CreatePaymentRequest request) {
        // 1. 校验订单
        Order order = orderService.requireByOrderNo(request.getOrderNo());
        if (order.getStatus() != OrderStatus.UNPAID.getCode()) {
            throw new BusinessException(ErrorCode.ORDER_STATUS_ERROR, "订单状态错误");
        }
        
        // 2. 幂等性检查：如果已存在支付记录，直接返回
        PaymentRecord existing = paymentRecordMapper.selectOne(
            new LambdaQueryWrapper<PaymentRecord>()
                .eq(PaymentRecord::getOrderNo, request.getOrderNo()));
        if (existing != null) {
            return toResponse(existing);  // 幂等：不重复创建
        }
        
        // 3. 创建支付记录
        PaymentRecord record = new PaymentRecord();
        record.setOrderNo(order.getOrderNo());
        record.setPaymentNo(BizNoUtil.paymentNo());  // "PAY202607010001"
        record.setAmount(order.getPayAmount());
        record.setChannel(request.getChannel());
        record.setStatus(PaymentStatus.INIT.getCode());  // 初始化
        paymentRecordMapper.insert(record);
        
        return toResponse(record);
    }

    @Transactional
    public PaymentResponse callback(PaymentCallbackRequest request) {
        // 1. 查询支付记录
        PaymentRecord record = paymentRecordMapper.selectOne(
            new LambdaQueryWrapper<PaymentRecord>()
                .eq(PaymentRecord::getPaymentNo, request.getPaymentNo()));
        
        // 2. 幂等性检查：如果已支付成功，直接返回
        if (StringUtils.hasText(record.getCallbackNo())
                && record.getCallbackNo().equals(request.getCallbackNo())
                && record.getStatus() == PaymentStatus.SUCCESS.getCode()) {
            return toResponse(record);  // 幂等：不重复处理
        }
        
        // 3. 更新支付状态
        record.setCallbackNo(request.getCallbackNo());
        record.setTradeNo(request.getTradeNo());
        record.setStatus(PaymentStatus.SUCCESS.getCode());
        paymentRecordMapper.updateById(record);
        
        // 4. 触发订单状态流转
        orderService.onPaid(record.getOrderNo());
        
        return toResponse(record);
    }

    @Transactional
    public PaymentResponse mockPay(Long id) {
        PaymentRecord record = requirePayment(id);
        
        // 幂等：已支付成功直接返回
        if (record.getStatus() == PaymentStatus.SUCCESS.getCode()) {
            return toResponse(record);
        }
        
        // 模拟支付回调
        String callbackNo = "MOCK-" + UUID.randomUUID().toString().replace("-", "").substring(0, 16);
        PaymentCallbackRequest callback = new PaymentCallbackRequest();
        callback.setPaymentNo(record.getPaymentNo());
        callback.setCallbackNo(callbackNo);
        callback.setTradeNo("MOCK_TRADE_" + record.getPaymentNo());
        
        return callback(callback);  // 调用回调逻辑
    }
}
```

**技术亮点**：
1. **幂等性设计**：
   - 创建支付时检查是否已存在支付记录，避免重复创建
   - 支付回调时检查 `callbackNo` 和状态，避免重复处理
   - 模拟支付时检查支付状态，避免重复支付
   
2. **状态流转触发**：支付成功后自动调用 `orderService.onPaid()`，订单状态从 `UNPAID` 流转到 `PAID`

3. **测试环境支持**：提供 `mockPay` 接口，无需真实第三方支付即可测试完整流程

```
     ▼                      ▼                      ▼                      ▼
```

**订单状态机实现（OrderStateMachine.java）**：

```java
public final class OrderStateMachine {

    // 校验状态流转是否合法
    public static void validateTransition(OrderStatus from, OrderStatus to) {
        boolean allowed = false;
        switch (from) {
            case UNPAID:
                // 未支付 → 已支付 或 已取消
                allowed = to == OrderStatus.PAID || to == OrderStatus.CANCELLED;
                break;
            case PAID:
                // 已支付 → 待发货 或 退款中
                allowed = to == OrderStatus.TO_SHIP || to == OrderStatus.REFUNDING;
                break;
            case TO_SHIP:
                // 待发货 → 已发货 或 退款中
                allowed = to == OrderStatus.SHIPPED || to == OrderStatus.REFUNDING;
                break;
            case SHIPPED:
                // 已发货 → 已完成 或 退款中
                allowed = to == OrderStatus.COMPLETED || to == OrderStatus.REFUNDING;
                break;
            case COMPLETED:
                // 已完成 → 退款中（售后）
                allowed = to == OrderStatus.REFUNDING;
                break;
            case CANCELLED:
            case REFUNDING:
                // 已取消/退款中：终态，不允许流转
                allowed = false;
                break;
        }
        
        if (!allowed) {
            throw new BusinessException(ErrorCode.ORDER_STATUS_ERROR,
                "Invalid order status transition: " + from.name() + " -> " + to.name());
        }
    }

    // 判断是否可取消
    public static boolean canCancel(OrderStatus status) {
        return status == OrderStatus.UNPAID;  // 仅未支付可取消
    }

    // 判断是否可发货
    public static boolean canShip(OrderStatus status) {
        return status == OrderStatus.TO_SHIP;  // 仅待发货可发货
    }

    // 判断是否可确认收货
    public static boolean canConfirm(OrderStatus status) {
        return status == OrderStatus.SHIPPED;  // 仅已发货可确认
    }
}
```

**技术亮点**：
1. **状态机设计**：
   - 使用 `validateTransition` 方法严格校验状态流转合法性
   - 每个状态只能流转到指定的目标状态
   - 防止非法状态流转（如：已发货直接变为已完成）
   
2. **终态保护**：
   - `CANCELLED` 和 `REFUNDING` 为终态，不允许再流转
   
3. **业务规则封装**：
   - `canCancel`、`canShip`、`canConfirm` 方法封装业务规则
   - Controller层直接调用，无需关心具体规则

4. **性能**：无需每次查询数据库验证session

### 3.1 技术选型问题

#### Q1: 为什么选择Spring Boot而不是传统SSM框架？

**回答**：

选择Spring Boot主要基于以下考虑：

2. **内嵌服务器**：
   - 传统SSM需要外部安装Tomcat、打包WAR文件、部署到Tomcat
   - Spring Boot内嵌Tomcat，打包JAR文件，直接 `java -jar` 运行
   
3. **开发效率**：
   - Spring Boot提供大量starter依赖，如 `spring-boot-starter-web`、`spring-boot-starter-data-jpa`
   - 无需手动管理依赖版本，避免版本冲突
   
4. **生态支持**：
   - Spring Boot完美整合MyBatis Plus、Redis、JWT等现代技术
   - Spring Cloud微服务组件（Gateway、Eureka、Feign）基于Spring Boot

**示例对比**：

#### Q2: 为什么选择Vue 3而不是Vue 2？

**回答**：

选择Vue 3主要基于以下优势：

1. **Composition API**：
   - Vue 2的Options API（data、methods、computed分离）不适合复杂组件
   - Vue 3的Composition API（`<script setup>` + ref/computed）可按逻辑组织代码，复用性更好
   
2. **性能提升**：
   - Vue 3使用Proxy响应式系统，性能优于Vue 2的Object.defineProperty
   - Vue 3优化虚拟DOM diff算法，渲染速度更快
   
3. **TypeScript支持**：
   - Vue 3源码使用TypeScript，类型支持更好
   - Vue 2需要额外配置才能支持TypeScript
   
4. **Vite构建工具**：
   - Vue 3配套Vite构建工具，开发环境秒级启动，热更新极快
   - Vue 2配套Webpack，启动和热更新较慢

#### Q3: 为什么使用JWT而不是Session？

**回答**：

选择JWT主要基于以下考虑：

1. **无状态**：
   - Session需要在服务端存储用户登录状态（内存或Redis）
   - JWT自包含用户信息（userId、role），无需服务端存储，适合微服务架构
   
2. **跨服务共享**：
   - 多个微服务可共享同一JWT Token，无需共享Session存储
   - Session在微服务架构中需要额外的Session共享方案（Spring Session + Redis）
   
3. **性能**：
   - JWT无需每次请求查询Session存储，减少数据库/Redis访问
   - Session每次请求都需要查询Session存储验证登录状态
   
4. **移动端友好**：
   - JWT适合移动端（App、小程序）认证，无需Cookie
   - Session依赖Cookie，移动端处理复杂

**JWT vs Session对比**：

| 对比项 | Session | JWT |
|--------|---------|-----|
| 存储位置 | 服务端（内存/Redis） | 客户端（localStorage） |
| 跨服务共享 | 需要Session共享方案 | 无需共享，直接验证Token |
| 性能 | 每次请求查询存储 | 无需查询，解析Token即可 |
| 移动端支持 | 依赖Cookie，复杂 | 无需Cookie，友好 |
| 安全性 | 存储在服务端，相对安全 | Token泄露风险，需配合HTTPS |

### 3.3 功能实现问题

#### Q7: 购物车是如何实现的？为什么用Redis而不是MySQL？

**回答**：

购物车使用Redis Hash结构实现，主要考虑如下：

1. **性能优势**：
   - Redis内存存储，读写速度远快于MySQL
   - 购物车操作频繁（添加、修改、删除），需要高性能
   
2. **临时数据**：
   - 购物车数据是临时数据，用户可能随时清空
   - 不需要持久化到MySQL，Redis更适合临时数据
   
3. **灵活更新**：
   - Redis Hash可单独更新某个商品的数量或选中状态
   - MySQL需要更新整行记录，效率较低
   
4. **用户隔离**：
   - 每个用户独立购物车（`cart:{userId}`）
   - Redis天然支持Key隔离，无需额外设计

**Redis Hash结构**：

```
Key: cart:123

Field: 1         Field: 5         Field: 8
Value: {         Value: {        Value: {
  "quantity":2,   "quantity":1,   "quantity":3,
  "selected":true "selected":false "selected":true
}               }               }
```

**代码实现**：

```java
// 添加商品到购物车
public void addItem(CartItemRequest request) {
    Long userId = requireUserId();
    String key = "cart:" + userId;
    String field = String.valueOf(request.getProductId());
    
    CartEntry entry = new CartEntry();
    entry.setQuantity(request.getQuantity());
    entry.setSelected(true);
    
    redisTemplate.opsForHash().put(key, field, toJson(entry));
}

// 获取购物车列表
public List<CartItemResponse> list() {
    Long userId = requireUserId();
    Map<Object, Object> entries = redisTemplate.opsForHash().entries("cart:" + userId);
    
    List<CartItemResponse> items = new ArrayList<>();
    for (Map.Entry<Object, Object> entry : entries.entrySet()) {
        Long productId = Long.valueOf(entry.getKey().toString());
        CartEntry cartEntry = parseEntry(entry.getValue().toString());
        
        // 从MySQL查询商品详情（标题、价格、库存）
        Product product = productService.requireProduct(productId);
        
        CartItemResponse item = new CartItemResponse();
        item.setProductId(productId);
        item.setTitle(product.getTitle());
        item.setPrice(product.getPrice());
        item.setQuantity(cartEntry.getQuantity());
        item.setSelected(cartEntry.getSelected());
        items.add(item);
    }
    return items;
}
```

#### Q8: 订单状态机是如何设计的？为什么要用状态机？

**回答**：

订单状态机是核心业务规则，设计如下：

**状态定义**：
- `UNPAID`（未支付）：订单刚创建，等待支付
- `PAID`（已支付）：支付成功，等待商户发货
- `TO_SHIP`（待发货）：商户准备发货
- `SHIPPED`（已发货）：已发货，等待用户确认收货
- `COMPLETED`（已完成）：用户确认收货，交易完成
- `CANCELLED`（已取消）：用户取消订单（仅未支付可取消）
- `REFUNDING`（退款中）：申请退款

**合法流转路径**：
```
UNPAID → PAID（支付成功）
UNPAID → CANCELLED（用户取消）
PAID → TO_SHIP（商户接单）
PAID → REFUNDING（用户退款）
TO_SHIP → SHIPPED（商户发货）
TO_SHIP → REFUNDING（商户退款）
SHIPPED → COMPLETED（用户确认收货）
SHIPPED → REFUNDING（用户退款）
COMPLETED → REFUNDING（售后退款）
```

**为什么要用状态机？**

1. **业务规则集中**：
   - 所有状态流转规则集中在 `OrderStateMachine` 类
   - 避免业务规则散落在各个Service方法
   
2. **防止非法操作**：
   - 状态机严格校验状态流转合法性
   - 防止非法操作（如：已发货直接变为已完成）
   
3. **代码清晰**：
   - Service层只需调用 `OrderStateMachine.validateTransition(from, to)`
   - 无需关心具体状态流转规则
   
4. **易于维护**：
   - 新增状态或流转规则只需修改状态机代码
   - 无需修改Service层代码

**代码示例**：

```java
// 商家发货：状态机校验
public OrderResponse ship(Long orderId, String logisticsNo) {
    Order order = requireOrder(orderId);
    
    // 状态机校验：必须为待发货状态
    if (!OrderStateMachine.canShip(OrderStateMachine.of(order.getStatus()))) {
        throw new BusinessException(ErrorCode.ORDER_STATUS_ERROR, "订单状态错误");
    }
    
    // 更新状态：TO_SHIP → SHIPPED
    OrderStateMachine.validateTransition(
        OrderStateMachine.of(order.getStatus()), 
        OrderStatus.SHIPPED
    );
    
    order.setStatus(OrderStatus.SHIPPED.getCode());
    order.setLogisticsNo(logisticsNo);
    orderMapper.updateById(order);
    
    return toResponse(order);
}
```

#### Q9: 支付幂等性是如何实现的？

**回答**：

支付幂等性通过以下方式实现：

1. **创建支付时的幂等性检查**：
   - 查询数据库是否已存在该订单的支付记录
   - 如果已存在，直接返回已有记录，不重复创建
   
2. **支付回调时的幂等性检查**：
   - 检查 `callbackNo`（回调编号）是否已处理
   - 如果 `callbackNo` 相同且状态为 `SUCCESS`，直接返回，不重复处理
   
3. **模拟支付时的幂等性检查**：
   - 检查支付状态是否已为 `SUCCESS`
   - 如果已支付成功，直接返回，不重复支付

**代码实现**：

```java
@Transactional
public PaymentResponse callback(PaymentCallbackRequest request) {
    PaymentRecord record = paymentRecordMapper.selectOne(
        new LambdaQueryWrapper<PaymentRecord>()
            .eq(PaymentRecord::getPaymentNo, request.getPaymentNo()));
    
    // 幂等性检查：如果已支付成功，直接返回
    if (StringUtils.hasText(record.getCallbackNo())
            && record.getCallbackNo().equals(request.getCallbackNo())
            && record.getStatus() == PaymentStatus.SUCCESS.getCode()) {
        return toResponse(record);  // 不重复处理
    }
    
    // 更新支付状态、触发订单状态流转
    record.setCallbackNo(request.getCallbackNo());
    record.setStatus(PaymentStatus.SUCCESS.getCode());
    paymentRecordMapper.updateById(record);
    
    orderService.onPaid(record.getOrderNo());
    
    return toResponse(record);
}
```

**幂等性作用**：

1. **防止重复扣款**：支付回调可能重复触发（网络超时重试），幂等性避免重复处理
2. **数据一致性**：保证支付状态与订单状态一致，不会出现支付成功但订单未更新
3. **分布式友好**：幂等设计适合分布式系统，可配合分布式锁进一步保障
